# 教科书深度解析 SKILL

## 工作流程

### 第一阶段：扫描章节

读取当前 skill 所在目录下的 `references/agent-prompts/chapter-scanner.md` 文件，调用 Agent 扫描书籍目录。

- name: "扫描章节"
  agent:
    model: "default"
    prompt: |
      {{read "references/agent-prompts/chapter-scanner.md"}}

      ## 当前任务
      请扫描以下书籍目录，识别所有章节文件：
      - 书籍目录: {{book_directory}}
      - 处理范围: {{chapters}}

      使用 Bash 工具列出目录内容，然后解析所有章节文件信息。
      返回符合指定格式的 JSON 结果。

### 第二阶段：创建输出目录

如果 `output_directory` 未指定，使用默认路径：`{{book_directory}}/deep-dive/`

使用 Bash 工具创建目录结构：
- `{{output_directory}}/`
- `{{output_directory}}/章节/` - 存放各章节的详细分析

### 第三阶段：Agent 并行处理章节（核心流程）

**重要：采用"每章一 Agent"的并行处理策略，每个 Agent 独立完成一整章的全部生成任务。**

**并行处理原则：**
- **每个 Agent 独占一章**：一个 Agent 只能处理某一章（Chapter）的内容，不可跨章处理
- **全章任务内聚**：单个 Agent 负责读取该章全部内容，并完成概览、小节、习题、质检全流程
- **章间并行执行**：不同章节的 Agent 可以同时运行，互不干扰

#### 3.1 为每章创建独立 Agent（并行执行）

为每个待处理章节创建一个 Agent，并行启动：

```yaml
# 为每个章节创建一个独立的 Agent 任务
agents:
  - name: "处理-第{{chapter.num}}章"
    model: "default"
    prompt: |
      你是一个专业的教材深度解析专家。你的任务是独立完成第 {{chapter.num}} 章的全部学习材料生成工作。

      ## 你的职责范围（仅限第 {{chapter.num}} 章）
      - **禁止处理其他章节**，专注于当前分配的章节
      - 完成该章从内容读取到质量检查的全流程

      ## 执行步骤

      ### 步骤 1：读取章节全部内容
      使用 Read 工具读取第 {{chapter.num}} 章的原始 markdown 文件：
      - 文件路径: {{book_directory}}/第{{chapter.num}}章*.md 或类似路径

      ### 步骤 2：生成章节概览
      读取模板 `references/chapter-overview-template.md`，生成：
      - 文件: `{{output_directory}}/章节/第{{chapter.num}}章_{{chapter.title}}/00_概览.md`
      - 包含：学习目标、本章速览、难度预警、前置知识、节依赖图、定理清单、核心逻辑线索、核心要点速查、概念对比表、定理依赖图、常见误解澄清、本章测验、快速复习卡、扩展阅读

      ### 步骤 3：逐一顺序生成小节学习材料
      **重要：只处理一级小节（如 {{chapter.num}}.1、{{chapter.num}}.2），不处理二级子节**

      1. 识别章节中所有一级小节（格式：`## X.Y 标题` 或 `### X.Y 标题`）
      2. 读取模板 `references/section-template.md` 和 `references/agent-prompts/section-analyzer.md`
      3. **按顺序逐个生成**每个小节的学习材料：
         - {{chapter.num}}.1 → 生成文件 `{{chapter.num}}.1_{{section1.title}}.md`
         - {{chapter.num}}.2 → 生成文件 `{{chapter.num}}.2_{{section2.title}}.md`
         - ...以此类推
      4. 每个小节文件应包含：背景动机、核心概念、定理/公式、详细证明、示例、可视化、常见陷阱、与其他节联系

      ### 步骤 4：生成习题与解答
      读取模板 `references/exercises-template.md`，生成：
      - 文件: `{{output_directory}}/章节/第{{chapter.num}}章_{{chapter.title}}/习题与解答.md`
      - 包含：基础练习题、综合练习题、挑战练习题、自测模拟卷
      - 每道题包含：难度标识、涉及知识点、逐步提示、详细解答、考点解析

      ### 步骤 5：质量检查与修复
      读取检查清单 `references/agent-prompts/content-checker.md`，对生成的所有文件进行自检：

      **检查清单：**
      1. [ ] 概览文件 (00_概览.md) 存在且内容完整
      2. [ ] 所有一级小节文件存在且命名规范（无二级子节文件）
      3. [ ] 习题与解答文件存在且内容完整
      4. [ ] 数学公式格式正确（行内 $...$，独立 $$...$$）
      5. [ ] Mermaid 图表语法正确
      6. [ ] 术语在全章保持一致

      **如发现质量问题：**
      - 立即修复有问题的文件
      - 重新生成不符合要求的内容
      - 确保所有检查项通过

      ## 上下文优化策略
      - 每完成一个小节生成后，**主动压缩上下文**，只保留该小节的关键摘要
      - 避免在上下文中堆积大量已完成的完整内容
      - 使用文件存储中间结果，而非长期保存在对话中

      ## 输出要求
      完成所有步骤后，返回简洁的报告：
      ```
      第 {{chapter.num}} 章处理完成：
      - 概览文件: 已生成
      - 小节文件: X 个（列出编号）
      - 习题文件: 已生成
      - 质量状态: 已通过自检 / 需人工复核
      ```

parallel: true
```

#### 3.2 监控 Agent 执行状态

并行启动所有章节的 Agent 后，监控执行状态：

```yaml
- name: "监控章节处理进度"
  action: |
    检查每个章节的 Agent 是否完成：
    - 已完成的章节：收集返回报告
    - 失败的章节：记录错误原因，标记为重试
    - 超时的章节：评估是否需要拆分任务
```

#### 3.3 收集各章结果

所有 Agent 完成后，汇总各章处理结果：

```yaml
- name: "汇总处理结果"
  action: |
    汇总所有章节的 Agent 报告：
    1. 统计成功/失败的章节数
    2. 收集质量检查报告
    3. 标记需要人工复核的章节
    4. 生成处理摘要
```

#### 3.4 失败章节重试（如需要）

对于处理失败或质量检查未通过的章节：

```yaml
agents:
  - name: "重试-第{{chapter.num}}章"
    condition: "chapter.status == 'failed'"
    model: "default"
    prompt: |
      重新处理第 {{chapter.num}} 章，上次失败原因：{{chapter.error}}

      请特别注意：
      1. 简化处理逻辑，避免上下文溢出
      2. 分更小的批次生成内容
      3. 增加自检频率

      [重复 3.1 的执行步骤]
```

**小节识别规则（Agent 内部执行时遵循）：**
1. 识别格式：`## X.Y 标题` 或 `### X.Y 标题`
2. 编号格式：`X.Y`（如 1.1, 1.2, 2.1）
3. **忽略** `X.Y.Z` 格式的二级子节（如 1.1.2, 1.2.3）
4. 二级子节的内容合并到所属一级小节中统一处理

### 第四阶段：生成书籍总览

所有章节处理并检查通过后，读取 `references/agent-prompts/book-indexer.md` 和 `references/book-overview-template.md`，调用 Agent 生成书籍总览：

- name: "生成书籍总览"
  agent:
    model: "default"
    prompt: |
      {{read "references/agent-prompts/book-indexer.md"}}

      ### 总览模板
      {{read "references/book-overview-template.md"}}

      ### 书籍信息
      - 书籍目录: {{book_directory}}
      - 书籍名称: {{book_title}}

      ### 章节摘要
      {{chapters_summary}}

      请生成完整的书籍总览和索引。

写入文件：
- `{{output_directory}}/00_书籍总览.md`
- `{{output_directory}}/索引.md`

### 第五阶段：最终质量验证

**最终检查：对生成的完整输出进行最终验证**

```yaml
- name: "最终质量验证"
  agent:
    model: "default"
    prompt: |
      请对以下完整输出进行最终质量验证：
      - 输出目录: {{output_directory}}

      ## 验证清单

      ### 全局结构
      - [ ] 书籍总览文件存在且内容完整
      - [ ] 导航文件存在且所有链接正确

      - [ ] 章节目录结构正确（每章包含概览、小节、习题）

      ### 章节一致性
      - [ ] 各章节编号连续无遗漏
      - [ ] 章节间引用关系正确
      - [ ] 术语在全书中保持一致
      - [ ] 各章仅包含一级小节分析（无二级子节文件）

      ### 格式统一性
      - [ ] 所有文件使用统一的标题层级
      - [ ] 表格格式统一
      - [ ] 代码块和公式样式一致

      ## 输出
      返回验证报告，包含：
      1. 验证结果（通过/失败）
      2. 发现的问题列表
      3. 修复建议
```

## 性能优化策略

### 并行度控制
- 默认同时运行最多 **2-4 个** Agent（根据模型限制调整）
- 如果章节数量超过并行度，分批处理
- **每章一 Agent**：每个 Agent 只负责一整章，不拆分过细

### 上下文管理（关键优化）

**及时压缩上下文：**
- 每个小节生成完成后，Agent 应主动清理上下文，只保留关键摘要
- 避免在对话历史中堆积大量已完成的完整内容
- 使用文件系统存储中间结果，而非长期保存在对话上下文中

**上下文优化技巧：**
1. **分段处理**：章节内容过长时，Agent 内部按小节分段读取和处理
2. **结果落地**：每生成一个文件立即写入磁盘，然后从上下文中移除原文
3. **摘要替代**：已完成的小节用摘要替代全文保留在上下文中
4. **定期重置**：完成重要阶段后可重置上下文，只保留必要的状态信息

**示例上下文压缩流程：**
```
生成小节 1.1 → 写入文件 → 保存摘要 "1.1: 核心概念..." → 移除原文
生成小节 1.2 → 写入文件 → 保存摘要 "1.2: 定理证明..." → 移除原文
...
```

### 错误恢复
- 单个章节处理失败不影响其他章节
- 记录失败的章节，最后统一报告
- 支持对失败章节单独重试
- 失败重试时采用更保守的上下文使用策略

## 注意事项

1. **Agent 职责边界**: 每个 Agent 严格只处理**一章**内容，禁止跨章处理。第 k 章的 Agent 只读写第 k 章的相关文件
2. **小节层级控制**: 严格只生成一级小节（X.Y）的学习材料，二级子节（X.Y.Z）内容合并到一级小节中处理
3. **上下文及时压缩**: Agent 在完成每个小节后必须主动清理上下文，避免上下文溢出。使用文件存储而非对话存储中间结果
4. **并行安全**: 各 Agent 只写入自己的章节目录，避免文件冲突
5. **资源控制**: 监控 API 调用频率，避免触发限制
6. **文件命名**: 章节目录和文件使用中文命名，保持与原始文件一致
7. **公式格式**: 所有数学公式使用 LaTeX 格式
8. **图片处理**: 保留原始内容中的图片引用，确保相对路径正确
9. **增量处理**: 如果输出目录已存在，询问用户是否覆盖或跳过已处理章节
10. **质量优先**: 宁可少处理几个章节，也要确保生成内容的质量
11. **自检机制**: 每个 Agent 在提交结果前必须进行自检，发现问题立即修复，不依赖外部检查
